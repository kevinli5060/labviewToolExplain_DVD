LabVIEW实时控制(NI-DAQmx)

LabVIEW实时控制(NI-DAQmx)范例项目使用NI-DAQmx代码实现确定性的、基于软件的设备控制。控制算法在实时控制器上运行。控制器从DAQ输入通道读取数据并写入数据至DAQ输出通道。通过运行在台式机上的用户界面发送命令和设置点更改至控制算法。实时控制器还用于监视应用的状态，例如，CPU负载和内存使用量。

该范例项目是基于“简单状态机”和“队列消息处理器”模板。关于模板工作的详细信息，可通过新建项目对话框获取“简单状态机”和“队列消息处理器”模板及文档。

功能

  • 确定性和基于软件的控制－实时控制器在超过1 kHz的频率下执行控制算法。
  • 用户界面－用户界面VI与实时控制器交互并显示控制循环的数据。该VI可随时连接或断开与设备的连接，且不会影响控制循环。
  • 错误处理－应用报告并记录实时控制器的全部错误。设备在遇到关键性错误时将关闭。

概述

范例项目由跨2个执行终端的8个并行循环组成。下列循环在台式机上并行运行：

  • 处理用户界面事件（UI Main.vi - 事件处理循环)－根据前面板事件生成发送至“UI消息循环”的消息。
  • 处理用户界面和实时控制器信息（UI Main.vi - UI消息循环）－使用网络流接收和响应来自事件处理循环和RT消息处理循环的消息。
  • 显示来自实时控制器的消息和数据（UI Main.vi - 监控循环）－显示RT循环采集到的信息的最新值－System Health and FPGA Monitoring.vi

下列循环在实时控制器上并行运行：

  • 控制设备（RT循环 - RSI Control.vi）读取DAQ输入通道、对每个通道应用控制算法并写入DAQ输出通道。控制算法使用设定点数据和发送自用户界面的PID增益。
  • 处理来自用户界面的命令（RT循环- UI Commands.vi）－使用网络流读取发送自UI Main.vi的命令，并生成相应的消息。
  • 处理来自实时控制器的全部循环的消息（RT Main.vi - 消息处理循环）－处理运行在实时控制器上的所有循环的消息。
  • 确保RT控制器保持响应（RT循环－Watchdog.vi）－发送信号至看门狗，确保RT控制器保持响应。
  • 监视实时控制器的诊断信息（RT循环－System Health and Monitoring.vi）－监控实时控制器的CPU和内存使用量及控制算法信息。该信息被写入网络发布的共享变量并出现在UI Main.vi的数据监控和系统监控选项卡。

运行范例项目

  1. 确保全部设备均已配置并与开发计算机连至同一网络。
  2. 在项目浏览器窗口中，添加或找到RT终端至顶层项目项。
  3. 将默认RT终端中的下列项目项拖曳至在步骤2中添加的终端：
    • Error Handlers文件夹
    • Globals文件夹
    • RT Loops文件夹
    • Support VIs文件夹
    • Type Definitions文件夹
    • Deterministic Loop Variables.lvlib
    • Shared Variables.lvlib
    • RT Main.vi
  4. 删除不再具有关联VI的默认RT终端项目项。
  5. 自定义下列位置的代码，以与DAQ设备进行通信：
    • DAQ Close.vi
    • DAQ Initialize.vi
    • DAQ Read.vi
    • DAQ Write Safe Outputs.vi
    • DAQ Write.vi
    • DAQ Tasks.ctl
  6. 在项目浏览器窗口，运行RT PXI终端» Main.vi。该VI部署程序至DAQ设备。RT Main.vi现在运行在目标设备中。
  7. 打开并运行我的电脑»UI Main.vi。
  8. 在控制器地址文本框中输入RT控制器的IP地址，并单击连接。
  9. 连接设备后，单击运行控制（位于控制设置页面）或运行手动（位于手动设置页面）。
  10. 按照需求调整通道的值并查看数据监控页面的输出。直至定义控制算法前，全部控制输出值将读取0。

修改范例项目

定义控件算法

在RT循环中找到条件结构的控制(PID)子框图－DAQ Control.vi，更改子框图以应用控制算法。例如，可使用包含在NI LabVIEW PID & Fuzzy Logic工具包内的PID VI。默认情况下，该算法在全部输出通道上返回常量值0。在各个通道上可执行不同的算法。

尽管范例项目设计用于执行PID算法，但可修改范例项目以使用任意算法。

定义连接至DAQ设备的硬件的安全值

全局安全输出值－Configuration Options.vi，定义连接至DAQ设备输出通道的硬件的理论安全值。RT循环中条件结构的默认子框图（安全状态）－DAQ Control.vi写入该值至全部DAQ输出通道。默认情况下，安全状态的值为0。

配置应用

在项目浏览器窗口，打开我的电脑»Globals»Global - Configuration Options.vi并配置范例项目的设置。
