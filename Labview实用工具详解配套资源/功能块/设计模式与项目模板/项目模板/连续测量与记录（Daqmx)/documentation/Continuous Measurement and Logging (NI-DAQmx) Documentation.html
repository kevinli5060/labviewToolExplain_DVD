连续测量和记录 (NI-DAQmx)

连续测量和记录(NI-DAQmx)范例项目连续采集测量并将测量记录至磁盘。范例并行执行5个循环：

  • 事件处理(Main.vi)－事件处理循环基于前面板事件生成消息。例如，用户单击开始或设置。
  • 用户界面消息发送(Main.vi)－消息处理循环从事件处理循环接收消息，并发送消息至其他消息处理循环以作出响应。
  • 采集数据(Acquisition.lvlib:Acquisition Message Loop.vi)－连续采集数据的消息处理循环。
  • 记录数据(Logging.lvlib:Logging Message Loop.vi)－连续记录采集到的数据的消息处理循环。
  • 显示数据(Main.vi)－使用采集到的数据更新波形图表的While循环。

范例项目还带有设置对话框(Settings.lvlib)，可用于配置应用程序。

范例项目是基于“队列消息处理器”模板。关于模板工作原理的详细信息，见“队列消息处理器”模板及其文档。通过“创建项目”对话框可获取上述信息。

项目框图

使用案例

连续测量和记录(NI-DAQmx)范例项目设计用于需要有响应用户界面的连续测量应用程序。即即使当应用程序执行其他命令时，用户仍可点击界面按钮。

运行范例项目

  1. 在项目浏览器窗口打开并运行Main.vi。
  2. 单击开始。程序开始采集波形数据。
  3. 单击其他前面板按钮以查看范例项目。

自定义范例项目

如默认的记录动作不能满足用户应用程序的需求，可通过下列方式手动修改范例项目：

  • 如要指定数据记录的位置，运行Main.vi并单击设置，使用记录文件路径控件。默认情况下，模板记录数据至LabVIEW Data\Logged Data.tdms。其中LabVIEW Data为LabVIEW数据文件夹。
  • 如要指定数据记录的触发，运行Main.vi并单击设置，使用触发时记录控件。可指定时间或门限值来触发记录。也可通过Main.vi中的强制触发控件手动触发记录。
  • 如要更改数据记录机制，可更改Logging.lvlib:Logging Message Loop.vi。例如，修改该VI通过数据流采集网络或磁盘数据。默认情况下，模板使用通知器控制记录数据的事件。
  • 如要更改代码写入数据至磁盘的时间，可更改Logging.lvlib:Log Data.vi。例如，使用“导出波形至电子表格文件”或“写入电子表格文件”VI。
  • 如要更改或添加数据记录所需的路径和文件引用句柄，可更改Logging.lvlib:Logging Configuration.ctl。
  • 如要更改采集的数据类型，可打开Configure Hardware.vi，选择“DAQmx创建虚拟通道”VI的另一个实例。

如默认记录速度太慢，可使用DAQmx开始新文件和DAQmx配置记录VI。请参考VI帮助文档获取详细信息。
